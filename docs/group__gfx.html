◆ CommandBufferSpecialization

enum class pom::gfx::CommandBufferSpecialization
strong

#include <pomegranate/graphics/gfx/commandBuffer.hpp>

Hint for what queue the CommandBuffer should be constructed from, only applies in graphics APIs where that is applicable such as Vulkan. In addition to this, some hardware won't have a dedicated queue for a certain specialization and the command buffer will be constructed from a different more general queue.

Enumerator
GRAPHICS 

Command buffer should be constructed by the graphics queue. This allows the buffer to use all commands.

TRANSFER 

Command buffer should be constructed by the transfer queue. Regardless of what queue this command buffer is constructed from this limits the commands that can be used by this command buffer to only transfer commands.

GENERAL 

Equivelent to CommandBufferSpecialization::GRAPHICS.

◆ Format

enum class pom::gfx::Format
strong

#include <pomegranate/graphics/gfx/gfx.hpp>

Specifies the format or type of various graphics procedures. Specifies image format, used for textures, framebuffers, etc. Currently maps directly to vulkan's VkFormat however this may change in the future.
Normalized and scaled formats (_*NORM and _*SCALED) are integers sampled as floats, where values map to [-1, 1] (signed) or [0, 1] (unsigned).
Int formats (_*INT) are integers sampled as integers.
Float formats (_*FLOAT) are floats sampled as floats.
SRGB formats (_SRGB) are unsigned integers with R, G, and B in the sRGB colorspace sampled as floats, the A component is sampled as an unsigned integer if present.
